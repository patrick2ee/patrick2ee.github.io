Context

Marketing email production at scale runs on infrastructure, not vibes. Every campaign a manager sends is a thin layer over the same dozen structural patterns — hero blocks, CTAs, two-column layouts, accessible footers. The question was whether to keep rebuilding those patterns ad hoc or invest in a system. I picked the second path and spent six years iterating on it.

Evolution

Scroll horizontally through the timeline below.

2020 — 2022

Ad hoc HTML

Emails developed campaign-by-campaign with internal engineering resources and an external HTML coding contract. No reuse, high cost per send.

2023

Accessibility-first refactor

Reimagined the 15 most common modules as reusable code blocks with accessibility at the forefront — readable markup, responsive structures without image backgrounds, ARIA labeling, and proper alt text.

2023 — 2024

Templates & expansion

Standardized hero–body–footer templates for marketing emails. Library expanded to 50+ modules. Still required copy-pasting templated HTML and updating it manually with content and design.

2025

Iterable snippet system

Used the Iterable platform migration as an opportunity to rebuild the library as snippets — abstracting code blocks into a preset list of variables that update via simple inputs and keywords. Managers no longer need to look at code.

2026

AI-native pipeline

Introduced a Claude skill that translates Figma designs into paste-ready snippets — closing the design-to-production loop and dropping the build time per email by an order of magnitude.

Module → variable abstraction

Every element that needs to be malleable for builders (marketers) becomes a variable. Variables can be declared with either literal values injected into the backend code (shown in blue), or conditional selections that inform which segment of code is shown in the final email (shown in green).

Design elements that are controlled by Creative, like the primary button styles or the size of hero images, are standardized in the backend code so Lifecycle never needs to worry about breaking guidelines.

All snippets also contain a link variable for consistent application of URLs on all clickable elements within the snippet.

Snippet library

Three representative snippets from the production library. Each card shows the snippet name, a short description, and an excerpt of the underlying code — full source available on request. Click “View full code” to expand any excerpt.

hero_centered

Centered hero with headline, optional subhead, and a CTA — including conditional CTA icon variants (lock, lightning, house, or none) and a fallback disclaimer.

1vp_wrapped

Single value-prop card with rounded corners. Optional icon image, optional title (two size variants), and optional body copy — each section conditionally rendered. Text alignment is variable-driven.

footer

Brand footer with a horizontal divider and conditional rendering by email type. Transactional emails (PL tx, mortgage tx) render a stripped-down version; marketing/leads emails get an unsubscribe link, with the URL variable swapped depending on whether the recipient is a known user or anonymous lead.

Outcome

Today the library powers 100% of marketing email production. The role of a Lifecycle manager fundamentally shifted — from a copy-paste-and-edit-HTML role to a strategy-and-content role. The Claude integration extended that lever further: design handoff is now an automated pipeline, not a queue.
